Steve Agent Lifecycle Architecture

DaemonCraft Companion Mode — Bot → Heartbeat → Gateway → LLM → Action → World

Flujo Roto: Turno Largo

  • • Steve inicia turno con mc_build(action="fill")
  • • Bot empieza a colocar bloques uno por uno
  • • Si el bot se queda stuck, el turno SIGUE corriendo
  • • 33 iteraciones después = 15 minutos sin progreso
  • • El steer del gateway llega pero no aborta el turno
  • • Resultado: plan cancelado por timeout de 5 min

Flujo Correcto: Chat Interrupt

  • • Usuario escribe en chat → Bot WebSocket
  • • Agent Loop setea chat_event = True
  • • AIAgent ve _interrupt_requested → aborta turno
  • • Próximo turno procesa el chat del usuario
  • • Self-echo filter evita loop infinito
  • • Idle auto-relay OFF (no spam cada 60s)

Propuesta: Modo Reacción vs Ejecución

  • • Modo Reacción: max 3-5 iteraciones, 30s timeout
  • • Responde chat, stuck, eventos del mundo
  • • Modo Ejecución: bg tasks (bot autónomo)
  • • LLM planifica → delega a bg_goto / bg_collect
  • • Heartbeat cada 10s durante ejecución activa
  • • Si stuck → wake_up → reacción → corrección
