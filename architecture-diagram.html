HermesCraft / DaemonCraft — Full Architecture

Hermes → Gemma-Andy → Guardian → Mineflayer → Minecraft | Path B (embodied service) canonical per Mariano's design

Narrative Layer (Hermes)

  • • CompAII (DeepSeek v4-pro) — CLI agent + direct mc_* control
  • • Hermes Gateway — Telegram bridge, DaemonCraft platform adapter
  • • HermesPolicy — 5-layer wrapper (scope→ambiguity→decompose→normalize→tools)
  • • Path 0: direct mc_* tools; Path B: embodied_plan → Gemma-Andy

Body Orchestrator (Gemma-Andy)

  • • Gemma 4 E4B-it, QLoRA fine-tuned (~1000 episodes)
  • • 68 tools schema v2 — 43 with executor_supported=true
  • • JSON input: high_level_command + world_state (14 fields)
  • • JSON output: body_plan + checks + tool_calls + failure_policy + operational_risk

Execution Layer (Mineflayer)

  • • Bot Server — HTTP API (/action/*, /task/*) + WebSocket dashboard
  • • Guardian v0 — tool whitelist + allowed_tools enforcement
  • • Safety: actionInProgress guard, stuck detection, climb-out pillaring
  • • Minecraft: Purpur 1.21.1 via Docker (daemoncraft.service)

Bugs Fixed (Session 2026-05-16)

  • • Race condition: actionInProgress guard prevents watchdog from canceling in-flight /action requests
  • • Dispatcher filtering: per-turn allowed_tools enforcement (tool_not_allowed for out-of-set tools)
  • • move_away semantics: flee() now parses coordinate strings ("x,y,z") as from target
  • • Mine safety: sort by Y descending (top-first) + underground detection + auto pillaring to surface
