**Type:** task

## What Is Missing

- **Feature flag `GATEWAY_HANDLES_CHAT`** does not exist anywhere
- **Plan mutations from player chat** — Gateway NEVER calls `/plan` endpoint
- Gateway does not use plan epoch when mutating (because it doesn't mutate at all)
- Loop still runs full `run_conversation()` with chat-capable history — no clear ownership boundary

## Root Cause

The "commitment problem" is unsolved: gateway promises things in chat ("I'll build that"), but never writes to the plan. Loop discovers nothing new on heartbeat.

4. Add epoch-aware retry logic for plan writes
5. Ensure loop does NOT process player chat when flag is on
