# DC-108: Loop Embodiment Cleanup

**Status:** in_progress  
**Priority:** high  
**Type:** task

## What Was Done

- `_post_chat` calls for player messages removed from loop
- Fake chat injection (quest_event, blueprint_updated) removed — loop now interrupts or ignores
- `MC_ALWAYS_CHAT` env var removed entirely
- Loop uses `BODY.md` prompt via `build_system_prompt()`
- `mc_chat` tool suppressed via `DC_LOOP_MODE=1`
- `_api_post` timeout reduced from 300s to 15s (hotfix) to prevent loop freeze

## What Is Missing / Broken

- **Loop heartbeat runs, but turns take too long** — `max_iterations=80` causes 20+ tool calls per turn
- **Context accumulates within each turn** — `run_conversation()` keeps all tool_call history until turn ends
- **BODY.md prompt is too aggressive** — "continue your current activity" causes infinite tool chains
- **Gateway plan mutations not implemented** (blocked on DC-107)
- **Feature flags not wired** — no way to gate loop vs gateway behavior
- **Quest engine still in loop** — should emit events to gateway, not fake chat

## Critical Hotfixes Applied

- `_api_post` timeout: 300s → 15s (prevents 5-minute freezes)
- `reasoning_effort`: `xhigh` → `low` in profile config
